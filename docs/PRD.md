# PRD: 础码 (ChuMa) — 计算机科学学习智能助教助学平台

## Problem Statement

计算机科学专业的学生在学习过程中面临以下核心痛点：

1. **学习效率低下**：408考研、数据库原理等科目知识点繁多且相互关联，学生缺乏系统化的知识图谱来理解知识之间的逻辑关系，往往陷入"刷题而不理解"的困境。
2. **个性化指导缺失**：传统教学模式无法根据每位学生的知识掌握水平、薄弱环节和学习进度提供个性化的学习路径与题目推荐，导致"千人一面"的学习体验。
3. **错题归因困难**：学生做完题目后难以精准定位错误根因（是概念不清、方法不熟还是计算失误），缺乏针对性的巩固练习和错题归因分析。
4. **教师教学负担重**：教师在学情分析、作业批改、教学资源推荐等方面投入大量重复性劳动，难以高效地进行精准教学和学生管理。

## Solution

础码 (ChuMa) 是一个面向计算机科学学科的 AI 智能助教助学平台，通过"双引擎"架构（助学引擎 + 助教引擎）和 GNN 图神经网络推理能力，为学生提供个性化、伴随式的学习支持，为教师提供数据驱动的教学辅助，实现"AI深度赋能、人机高效协同"的新型教学科研范式。

首期平台聚焦两大核心学习场景：**408考研备考**与**数据库原理学习**，通过知识图谱可视化、GraphRAG 检索增强生成、动态题目推荐与指导等技术，帮助学生从"学知识"转变为"强能力"。算法刷题训练将在后续版本中支持。

## User Stories

### 学生侧（助学场景）

1. As a 计算机科学专业学生, I want to 通过知识图谱可视化探索408/数据库的学科知识体系, so that 我能直观理解各知识点之间的逻辑关系和依赖路径。
2. As a 备考408的学生, I want to 获得基于我当前知识水平的个性化学习计划, so that 我能高效分配复习时间，优先攻克薄弱环节。
3. As a 学习数据库原理的学生, I want to 在遇到SQL查询、范式判断等问题时获得分步引导式解题指导, so that 我不仅得到答案，还能理解解题思路。
4. As a 复习408的学生, I want to 系统根据我的错题自动识别我的薄弱知识点（如操作系统、计算机网络等）, so that 我能进行针对性练习。
5. As a 学习中的学生, I want to 通过自然语言向AI助教提问计算机科学专业问题, so that 我能获得准确、可追溯的知识解答。
6. As a 使用平台学习的学生, I want to 在个人学习中心看到我的学习进度仪表盘, so that 我能清晰了解自己的学习状态和目标完成度。
7. As a 复习错题的学生, I want to 查看错题本中每道错题的归因分析（概念错误/方法错误/计算错误）, so that 我能有针对性地进行知识巩固。
8. As a 使用知识图谱的学生, I want to 点击某个知识点后看到其关联的练习题、学习资源和前置知识, so that 我能进行关联式学习。
9. As a 学习新章节的学生, I want to 系统根据我的学习历史推荐下一阶段最适合学习的内容, so that 我的学习路径是动态优化的。
10. As a 使用平台的学生, I want to 接收智能提醒（如学习计划到期、薄弱知识点巩固建议）, so that 我能保持学习节奏不中断。
11. As a 备考临近考试的学生, I want to 获得模拟考试和知识点掌握度评估报告, so that 我能了解自己的备考状态并调整策略。
12. As a 学习数据库的学生, I want to 通过可视化工具理解SQL执行计划和查询优化过程, so that 我能深入理解数据库内部机制。
13. As a 遇到困难题目的学生, I want to 获得渐进式提示（先给思路方向，再给关键步骤，最后给完整解答）, so that 我能在有引导的情况下自主思考。
14. As a 长期使用平台的学生, I want to 查看我的学习历史趋势图（如每日做题量、正确率变化）, so that 我能直观感受到自己的进步。
15. As a 组队学习的学生, I want to 与同学共享学习计划和错题本, so that 我们能互相督促、共同进步。
16. As a 学习操作系统的学生, I want to 通过模拟器理解进程调度、内存管理等核心概念, so that 我能在实践中掌握理论知识。
17. As a 使用平台的学生, I want to 所有AI生成的知识解答都能标注权威来源（如教材章节、论文出处）, so that 我能验证答案的准确性。

### 教师侧（助教场景）

18. As a 计算机科学教师, I want to 在平台上管理我的课程内容（发布课件、布置作业、设置考试）, so that 我能高效组织教学活动。
19. As a 教师, I want to 查看所教班级的学情分析报告（知识点掌握分布、成绩趋势、薄弱环节统计）, so that 我能精准了解班级整体学习状况。
20. As a 教师, I want to 系统自动识别学习风险较高的学生并发出预警, so that 我能及时进行干预和辅导。
21. As a 教师, I want to 对学生的作业和考试进行批量智能评估, so that 我能大幅减少重复性批改工作。
22. As a 教师, I want to 基于学情数据获得教学策略优化建议（如哪些知识点需要重新讲解）, so that 我能进行数据驱动的精准教学。
23. As a 教师, I want to 为学生建立能力画像（编程能力、理论基础等维度）, so that 我能全面了解每位学生的能力发展情况。
24. As a 教师, I want to 基于GNN推荐引擎获得教案生成建议, so that 我能快速获得高质量的教学设计参考。
25. As a 教师, I want to 将学生按能力水平进行分组管理, so that 我能实施差异化教学策略。
26. As a 教师, I want to 查看每位学生的详细学习轨迹（学习时长、题目完成情况、知识掌握曲线）, so that 我能进行个性化指导。
27. As a 教师, I want to 系统根据教学目标自动推荐相关的教学资源（论文、案例、视频等）, so that 我能快速扩充教学素材。

### 平台通用场景

28. As a 新用户, I want to 通过注册登录快速创建个人账户并选择身份（学生/教师）, so that 我能开始使用平台功能。
29. As a 学生用户, I want to 在平台上以自然语言发起多轮连贯对话, so that 我能获得流畅的交互体验。
30. As a 用户, I want to 平台支持文本、代码、图表等多模态内容的输入和展示, so that 我能以多种方式与平台交互。
31. As a 使用平台的学生, I want to Monaco Editor 提供 SQL 语法高亮、自动补全和错误提示, so that 我能高效编写和调试 SQL 语句。
32. As a 用户, I want to 平台响应迅速、界面友好, so that 我的学习和教学过程不被技术问题打断。

## Implementation Decisions

### 模块划分

1. **用户系统模块**
   - 注册/登录/身份管理（学生、教师两种角色）
   - 基于 JWT 的身份认证与权限控制
   - 用户档案管理（学习偏好、学科选择等）

2. **前端界面模块（Vue）**
   - 采用 Vue 3 + Element Plus 构建响应式 UI
   - 集成 ECharts 实现知识图谱可视化、学情数据图表展示
   - 使用 Vue Router 实现单页应用路由管理

3. **应用服务层（FastAPI）**
   - RESTful API 设计，FastAPI 提供高性能异步接口
   - Pydantic 进行请求/响应数据校验
   - SQLAlchemy ORM 管理关系型数据
   - JWT 实现无状态认证

4. **助学智能引擎（学生侧）**
   - **GraphRAG（图检索增强生成）**：基于学科知识图谱进行语义检索，结合大模型生成准确、**可追溯**的知识解答
   - **知识推理**：基于图结构的关系学习，发现知识点之间的隐含关联
   - **学习计划生成**：根据学生当前知识水平和目标，动态生成个性化学习路径
   - **进度跟踪**：实时监控学习进度，提供智能提醒
   - **错题本分析**：自动分类错题归因（概念/方法/计算），生成薄弱点报告
   - **动态题目指导**：提供渐进式提示的解题引导（支持408选择题/综合题、数据库SQL题/设计题等）

5. **助教智能引擎（教师侧）**
   - **学情分析**：基于数据挖掘技术分析班级整体和个体学习数据
   - **教学优化**：根据学情数据推荐教学策略调整方案
   - **学生画像**：多维度能力评估（编程能力、算法思维、理论基础）
   - **预警机制**：自动识别学习风险学生，触发教师干预
   - **批量评估**：AI 辅助作业和考试的自动/半自动批改
   - **内容推荐**：基于教学目标和学情数据推荐教学资源

6. **GNN 推理模块（共享）**
   - 图神经网络构建学科知识图谱的向量表示（408四科：数据结构、计算机组成原理、操作系统、计算机网络；数据库原理）
   - 学生侧：基于学生知识状态和图谱结构进行题目推荐
   - 教师侧：基于学生掌握情况和知识图谱进行教案推荐生成

7. **功能模块层**
   - 助学：个人学习中心（学习仪表盘）、知识图谱探索、题目实战（408四科/数据库原理）
   - 助教：学生管理（班级/分组）、数据分析（学情报告）

8. **数据存储层**
   - **ChromaDB**：向量数据库，存储知识图谱嵌入和文档向量，支持语义检索
   - **Redis**：缓存热点数据、管理用户会话和学习状态
   - **OSS**：文件存储，存放课件、视频、图片等教学资源

### 关键技术决策

- 采用 **GraphRAG** 而非传统 RAG，因为计算机科学学科知识具有强关联性（数据结构→算法→操作系统→编译原理），图结构能更好地表达知识间的依赖关系
- **GNN 推理**作为共享引擎，避免学生侧和教师侧重复建模，同时能利用学生-题目-知识点的异构图谱进行更精准的推荐
- 前端集成 **Monaco Editor**（VS Code 同款编辑器），提供 SQL 语句编写与执行能力，支持数据库原理课程的在线实验与练习
- 使用 **ECharts** 实现知识图谱的交互式可视化，支持节点展开、路径高亮、关联查询等操作

## Testing Decisions

### 测试原则
- 只测试外部可观察的行为，不测试实现细节
- 优先测试用户核心路径（注册→选择学科→学习/练习→获得反馈）
- AI 生成内容的准确性需要与标准答案/权威来源进行对比验证

### 测试模块与策略

1. **用户系统测试**
   - 注册/登录流程的端到端测试
   - JWT 认证与权限控制的接口测试
   - 学生/教师角色隔离测试

2. **助学引擎测试**
   - GraphRAG 问答准确性测试：准备至少 3 个典型计算机科学问题（如408真题、SQL查询题、数据库设计题），验证生成答案与标准答案的一致性
   - 学习计划生成测试：验证不同知识水平输入是否生成差异化计划
   - 错题归因测试：验证错题分类的准确性
   - 题目推荐测试：验证推荐结果是否符合学生当前知识水平

3. **助教引擎测试**
   - 学情分析报告测试：使用模拟数据验证分析结果的合理性
   - 批量评估测试：对比 AI 评分与人工评分的一致性
   - 预警机制测试：验证风险学生识别的准确性

4. **前端交互测试**
   - 知识图谱可视化渲染与交互测试
   - Monaco Editor SQL 编辑与执行流程测试
   - 响应式布局兼容性测试

5. **API 接口测试**
   - RESTful API 的请求/响应格式验证
   - 并发性能测试（多用户同时使用场景）
   - 错误处理与边界条件测试

### 典型测试案例（内容质量验证）

根据赛事要求，需提供至少 3 个典型问题的测试案例：

| 编号 | 学科场景 | 测试问题 | 验证方式 |
|------|---------|---------|---------|
| TC-1 | 408考研-操作系统 | "请解释操作系统中银行家算法的执行过程，并判断以下资源分配是否安全" | 与标准教材答案对比，验证步骤完整性和结论正确性 |
| TC-2 | 数据库原理 | "给定关系模式R(A,B,C,D)和函数依赖集F={A→B, B→C, A→D}，判断R属于第几范式" | 与标准答案对比，验证范式判断依据和推理过程 |
| TC-3 | 408考研-计算机网络 | "请详细说明TCP三次握手的过程，并解释为什么不能使用两次握手" | 与标准教材答案对比，验证过程描述完整性和解释的准确性 |

## Out of Scope

1. **移动端适配**：首期版本仅支持 Web 端，移动端适配不在本期范围内。
2. **模型微调**：首期版本基于 RAG 和提示工程实现学科垂类能力，LoRA/SFT 微调作为后续优化方向。
3. **助研场景**：首期聚焦助教和助学两大场景，学术写作辅助、论文分析等助研功能不在本期范围内。
4. **算法刷题模块**：首期聚焦 408 考研和数据库原理两大场景，在线算法刷题（LeetCode 风格题目、代码编辑器评测、数据结构可视化等）将在后续版本中支持。
5. **多学科扩展**：首期聚焦计算机科学学科（408、数据库原理），其他学科（教育学、医学、法学等）的扩展不在本期范围内。
6. **社交功能**：学习社区、讨论区、师生互动等社交功能不在本期范围内。
7. **支付与商业化**：付费课程、会员体系等商业化功能不在本期范围内。

### 目标用户群体
- **主要用户**：计算机科学专业本科生、考研备考学生
- **教师用户**：高校计算机科学专业教师、辅导员
- **规模化验证目标**：50人以上班级试用（争取额外加分）

### 迭代规划
- **v1.0（赛事交付）**：核心助教+助学功能，知识图谱可视化，GraphRAG问答，错题分析，学情报告，聚焦 **408考研** 与 **数据库原理** 两大场景
- **v1.1（后续优化）**：模型微调（LoRA/SFT），助研场景支持，算法刷题模块（在线编码评测、数据结构可视化），移动端适配
- **v2.0（远期规划）**：多学科扩展，社交学习功能，商业化探索
